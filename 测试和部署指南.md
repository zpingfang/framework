# n8n工作流测试和部署指南

## 🚨 解决404错误问题

你遇到的错误提示：
```
{"code":404,"message":"This webhook is not registered for GET requests. Did you mean to make a POST request?"}
```

**原因**：你使用了GET请求访问了一个只接受POST请求的webhook。

## ✅ 正确的测试方法

### 1. 激活工作流
首先确保工作流已经**激活**：
- 在n8n界面中，点击工作流右上角的"激活"开关
- 确保状态显示为"Active"

### 2. 获取正确的Webhook URL
激活后，你会看到webhook节点显示真实的URL，格式如下：
```
https://your-n8n-domain:5678/webhook/generate-video
```

### 3. 使用正确的HTTP方法测试

#### 方法A：使用curl命令
```bash
# 基础测试
curl -X POST "https://your-n8n-domain:5678/webhook/generate-video" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://www.amazon.com/dp/B08N5WRWNW",
    "scene_count": 5,
    "video_style": "专业商务",
    "voice_language": "中文"
  }'

# 简化测试（只传URL）
curl -X POST "https://your-n8n-domain:5678/webhook/generate-video" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
```

#### 方法B：使用Postman
1. 创建新的POST请求
2. URL: `https://your-n8n-domain:5678/webhook/generate-video`
3. Headers: `Content-Type: application/json`
4. Body (raw JSON):
```json
{
  "url": "https://www.amazon.com/dp/B08N5WRWNW",
  "scene_count": 10,
  "video_style": "专业商务",
  "voice_language": "中文"
}
```

#### 方法C：使用Python脚本
```python
import requests
import json

url = "https://your-n8n-domain:5678/webhook/generate-video"
payload = {
    "url": "https://www.amazon.com/dp/B08N5WRWNW",
    "scene_count": 5,
    "video_style": "专业商务",
    "voice_language": "中文"
}

response = requests.post(url, json=payload)
print(f"状态码: {response.status_code}")
print(f"响应: {response.json()}")
```

## 🔧 部署前配置检查

### 1. API凭据配置
在n8n中添加以下凭据：

#### Hugging Face API
- 凭据类型：HTTP Header Auth
- 名称：`huggingface-api`
- Header名称：`Authorization`
- Header值：`Bearer YOUR_HF_TOKEN`

#### AI/ML API (Kling视频)
- 凭据类型：HTTP Header Auth
- 名称：`aiml-api`
- Header名称：`Authorization`
- Header值：`Bearer YOUR_AIML_API_KEY`

#### Segmind API (配音)
- 凭据类型：HTTP Header Auth
- 名称：`segmind-api`
- Header名称：`x-api-key`
- Header值：`YOUR_SEGMIND_API_KEY`

### 2. API密钥获取

#### Hugging Face (免费)
1. 访问：https://huggingface.co/settings/tokens
2. 创建新的Access Token
3. 权限选择：Read

#### AI/ML API (Kling视频)
1. 访问：https://aimlapi.com
2. 注册并获取API密钥
3. 有免费配额

#### Segmind (配音)
1. 访问：https://segmind.com
2. 注册并获取API密钥
3. 有免费试用

## 🚀 完整部署流程

### 1. Docker部署n8n
```bash
# 创建数据目录
mkdir -p ~/.n8n

# 启动n8n容器
docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  -e WEBHOOK_URL=https://your-domain:5678/ \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=your-password \
  n8nio/n8n:latest
```

### 2. 导入工作流
1. 登录n8n (http://your-domain:5678)
2. 点击 "Import from file" 或 "Import from URL"
3. 粘贴 `n8n_video_generator_workflow_complete.json` 内容

### 3. 配置凭据
- 依次配置所有API凭据
- 测试每个API连接

### 4. 激活工作流
- 点击右上角的"激活"开关

## ⚠️ 常见问题解决

### 问题1：404错误
- ✅ 确保使用POST方法
- ✅ 确保工作流已激活
- ✅ 检查URL路径正确

### 问题2：API配额限制
- 🔄 轮换使用不同的API服务
- ⏰ 等待配额重置
- 💰 考虑升级付费计划

### 问题3：生成失败
- 🔍 检查API密钥有效性
- 📱 测试网络连接
- 📋 查看详细错误日志
