# n8n工作流快速导入指南

## 问题解决

如果遇到"Could not import file"错误，请按照以下步骤操作：

### 方法1：使用修复后的JSON文件

使用 `n8n_video_generator_workflow_fixed.json` 文件，这个文件已经修复了所有JSON格式问题。

### 方法2：手动验证JSON格式

1. **复制JSON内容**
2. **在线验证**：访问 https://jsonlint.com/ 验证JSON格式
3. **检查常见问题**：
   - 确保所有字符串都用双引号包围
   - 检查是否有多余的逗号
   - 确保所有括号都正确闭合

### 方法3：分步导入（推荐）

如果完整工作流导入失败，可以分步创建：

#### 步骤1：创建基础节点
```json
{
  "name": "网页视频生成器-简化版",
  "nodes": [
    {
      "parameters": {
        "httpMethod": "POST",
        "path": "generate-video",
        "responseMode": "responseNode"
      },
      "id": "webhook-node",
      "name": "触发器",
      "type": "n8n-nodes-base.webhook",
      "typeVersion": 1,
      "position": [240, 300]
    },
    {
      "parameters": {
        "values": {
          "string": [
            {
              "name": "url",
              "value": "={{ $json.body.url }}"
            }
          ]
        }
      },
      "id": "set-node",
      "name": "设置参数",
      "type": "n8n-nodes-base.set",
      "typeVersion": 3.4,
      "position": [460, 300]
    }
  ],
  "connections": {
    "触发器": {
      "main": [
        [
          {
            "node": "设置参数",

          }

  }

```

#### 步骤2：逐步添加节点
导入基础版本后，在n8n界面中手动添加其他节点：

1. **HTTP Request** 节点 - 获取网页内容
2. **HTML Extract** 节点 - 提取网页信息  
3. **Code** 节点 - 内容分析
4. **HTTP Request** 节点 - AI服务调用

## 常见错误及解决方案

### 错误1: "Invalid JSON"
**原因**: JSON语法错误
**解决**: 使用在线JSON验证器检查格式

### 错误2: "Node type not found"
**原因**: n8n版本不兼容
**解决**: 更新n8n到最新版本

### 错误3: "Invalid node parameters"
**原因**: 节点参数格式错误
**解决**: 检查节点参数是否符合当前n8n版本

## 手动创建步骤

如果JSON导入完全失败，可以手动创建工作流：

### 1. 创建Webhook触发器
- 添加Webhook节点
- 设置HTTP方法为POST
- 设置路径为 "generate-video"

### 2. 添加参数设置节点
- 添加Set节点
- 设置以下字段：
  - url: `={{ $json.body.url }}`
  - scene_count: `={{ $json.body.scene_count || 10 }}`
  - video_style: `={{ $json.body.video_style || '专业商务' }}`

### 3. 添加网页抓取节点
- 添加HTTP Request节点
- URL: `={{ $node['Set'].json.url }}`
- 方法: GET

### 4. 连接节点
按顺序连接所有节点
